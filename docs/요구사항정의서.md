# PTI 프로젝트 요구사항 정의서

## 1. 개요
본 문서는 PTI(Pre-Trip Inspection) 데이터 관리 시스템의 요구사항을 정의합니다. 본 시스템은 컨테이너 PTI 현황을 효율적으로 관리하고 시각화하는 것을 목적으로 합니다.

## 2. 주요 기능 요구사항

### 2.1 대시보드 (Dashboard)
- 전체, 대기(Pending), 진행(In Progress), 완료(Pass), 상차(Picked Up) 상태별 통계 카드 제공.
- 실시간 데이터 현황 시각화.

### 2.2 PTI 목록 관리 (PTI List)
- 모든 PTI 레코드의 목록 표시.
- 검색 기능 (컨테이너 번호, 예약 번호 등).
- 상태별 필터링 기능.
- 개별 레코드 수정 및 삭제 기능.
- 대량 작업(삭제 등) 지원.

### 2.3 데이터 입력 및 편집 (Data Entry)
- 개별 PTI 레코드 추가 및 편집 폼 제공.
- **Bulk Paste (대량 붙여넣기)**: Excel 등에서 복사한 텍스트 데이터를 파싱하여 다량의 레코드를 한 번에 추가하는 기능.

### 2.4 데이터 임포트 (Data Import)
- 외부 Excel 파일(`SAMPLE.xlsx`)을 읽어 시스템 초기 데이터로 변환하는 유틸리티 제공.

### 2.5 보고서 (Reports)
- 기간별 PTI 처리 현황 및 작업 효율 통계 보고서 제공.

### 2.6 휴지통 (Trash)
- 삭제된 데이터를 임시 보관하고 복구하거나 영구 삭제하는 기능.

### 2.7 설정 (Settings)
- 이메일 알림 발송을 위한 서버 정보 및 수신자 규칙 설정.

## 3. 데이터 요구사항
- **저장 방식**: 로컬 SQLite 데이터베이스를 사용하여 데이터 영속성 확보.
- **데이터 구조**: 컨테이너 번호, 예약 번호, 선사, 고객사, 장소, 요청일, 상태, 온도, 환적 정보 등 포함.

## 4. 비기능 요구사항
- **보안**: 로컬 환경에서 실행되며 외부 클라우드 노출 최소화.
- **성능**: 대량 데이터(Bulk Paste) 처리 시 끊김 없는 사용자 경험 제공.
- **사용성**: 현대적이고 직관적인 다크 모드 기반 UI.
